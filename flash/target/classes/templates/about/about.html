Project structure

Welcome to Flash project structure for flask, in this topic we will explain how and why is made id. So lets see the project structure: 

                        
                    ~/Project     
                        |__ /app 
                            |__/controllers  #Determinate what happend and what response to send when someone make a request
                            |__/models       #Create your models or entities
                            |__/routes       #Write here the Blueprint routes
                            |__/service      #Do the stuff your models are allow to do
                            |__/static       #Store your static files like css, js, images...etc
                            |__/templates    #Store your html templates
                            |--__init_.py
                        |__/config 
                            |--config.py
                        |__/db
                        |__/migrations
                        |__/tests
                        |__/venv
                        |--app.py  
                        |--README.txt
                        |--requirements.txt

After this you may have some questions about the project and why we choose this structure. First of all, we have built this structure following the MVC design pattern. If have knowledge of this pattern you may know that your main work is under models, controllers and template(view) folders. OK lets start from the beginning and explain why we create all of this folders.

~/Project/app

Here we have our project itself from our models to our views, we assume you have knowledge about Flask (or any backend framework) and how it works

~/Project/app/__init__.py

Since we are using application factory to create our Flask application, here is where the create_app() method is placed, and where we do the required stuff in order to create our application object. For more information about Application Factory click here

~/Project/config

Config folder is where we put all our config files, this is the config file that we use: 

                            
import os

basedir = os.path.relpath("../db")

template_dir = os.path.realpath('app/templates')
static_dir = os.path.realpath('app/static')


class BaseConfig():
SECRET_KEY = os.environ.get('SECRET_KEY') or ' 12345' #change this secret key please XD
SQLALCHEMY_TRACK_MODIFICATIONS = False

class DevelopmentConfig(BaseConfig):
DEBUG = True
SQLALCHEMY_DATABASE_URI = os.environ.get('DEV_DATABASE_URL') or 'sqlite:///' + os.path.join(basedir, 'data-development.sqlite')

PROFILE = " * Developer profile activated"

class TestingConfig(BaseConfig):
TESTING = True
SQLALCHEMY_DATABASE_URI = os.environ.get('TEST_DATABASE_URL') or 'sqlite:///' + os.path.join(basedir, 'data-testing.sqlite')

PROFILE = " * Testing profile activated"

class ProductionConfig(BaseConfig):
SQLALCHEMY_DATABASE_URI = os.environ.get('DATABASE_URL') or 'sqlite:///' + os.path.join(basedir, 'data-production.sqlite')

PROFILE = " * Production profile activated"


config = {
'development': DevelopmentConfig,
'testing': TestingConfig,
'production': ProductionConfig,

'default': DevelopmentConfig
}

As you can see you have 3 config class : DevelopmentConfig, TestingConfig and ProductionConfig To choose one add a enviromment variable called 'FLASK_ENV' and one of the the next values development, testing or production

~/Project/db

All the SQLite files that can be created in your Flask application is placed here, dont worry beacouse we already configurate this project to reach this behaviour

~/Project/migrations

This folder is necesary to exist if you want to use SQL Alchemy otherwise feel free to remove it from your project

~/Project/tests

Yes, obviously this folder contain all your test files.

~/Project/app.py

OK this file is important because is the entry point of your application and where the application object itself is created

~/Project/--requirements.txt

This requirements.txt file is used for specifying what python packages are required to run the project you are looking at

~/Project/README.txt

This is a brief description of your application.
